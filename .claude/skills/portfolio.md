# Skill: Portfólio Público — GitHub como Vitrine

> Carregar quando o usuário mencionar publicação, README, GitHub ou portfólio.

---

## Princípio

GitHub não é depósito de código — é vitrine profissional. Três projetos bem documentados > trezentos exercícios de tutorial. O leitor decide em segundos se vale explorar o perfil.

## Curadoria de Repositórios

- **Arquivar:** repos de curso, exercícios, forks abandonados, projetos descartados.
- **Manter públicos** apenas repos que: resolvem problema real, seguem o fluxo do framework, possuem README conforme abaixo.
- **Fixar no máximo seis repos** — vitrine, não catálogo.

Para cada repo fixado: título orientado a problema (`crop-disease-detector`, não `project-final-v2`), descrição focada no domínio ("Reduz erro de previsão de safra", não "Built with TensorFlow"), tags de linguagem configuradas.

## README Obrigatório (inglês)

1. **Contexto de negócio.** Por que o sistema existe. Não o que o código faz — por que foi escrito.
2. **Diagrama de arquitetura.** Mermaid ou imagem. Pensamento em sistemas, não em arquivos.
3. **Decisões de engenharia.** Escolhas técnicas com justificativa. Fonte: seção Decisões Técnicas do `registry.md`.
4. **Como executar.** Setup em uma linha quando possível. Docker Compose para múltiplos serviços.

## README de Perfil (Bio)

- Posicionamento: `[papel] specialized in [domínio]. [resultado mensurável].`
- Métrica de impacto: números > adjetivos.
- Sem "I love coffee and coding".

## Histórico Visível

- Commits: Conventional Commits é posicionamento público.
- PRs: preenchidos via template, argumentação técnica visível.
- Issues: bem descritas mostram capacidade de comunicar problemas.

## Contribuições Open Source

Comece pelas libs que já usa. `good first issue` em repos internacionais. Correções de documentação contam. Um commit aceito em lib de uso amplo > dezenas de certificados.

## Checklist

- [ ] README de perfil com posicionamento e métrica.
- [ ] Max 6 repos fixados, títulos orientados a problema.
- [ ] READMEs em inglês com contexto, diagrama, decisões, execução.
- [ ] Repos de curso/exercício arquivados.
- [ ] Conventional Commits nos repos fixados.
- [ ] PRs públicos preenchidos via template.
- [ ] Ao menos uma contribuição open source.
