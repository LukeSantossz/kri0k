# kri0k - Project Instructions

## Overview

kri0k é um framework de engajamento de segurança ofensiva com agente LLM autônomo. Combina Rust (estado canônico, validação, auditoria) com Python (LangGraph, LLM orchestration).

## Architecture

```
Python (LangGraph) → PyO3 Bridge → Rust Core
     ↑                              ↓
  Proposals ←── JSON Snapshots ←── Graph State
```

**Key constraint:** LLM never executes directly. All actions pass through Rust scope validation.

## Crates

| Crate | Purpose |
|-------|---------|
| kri0k-core | Error types, NodeId/EdgeId, Ttp trait, AuditSink, Scope, Safeguards |
| kri0k-graph | petgraph wrapper, Node/Edge/NodeKind/EdgeKind |
| kri0k-pybridge | PyO3 cdylib, `_native` module, Tokio runtime |

## Code Style

### Rust

- Clippy strict: `unwrap_used`, `panic`, `unimplemented` are denied
- No `unsafe` code (workspace lint)
- `thiserror` for error types
- `serde` for all cross-boundary types

### Python

- ruff for linting (E, W, F, I, N, UP, ANN, S, B)
- mypy strict mode
- Type hints required for all public functions
- `_native.pyi` stubs for Rust bindings
